class Solution { public: int lengthOfLongestSubstring(string s) { int cnt128; int ans = 0, n = s.size(); for (int l = 0, r = 0; r < n; ++r) { ++cnt[s[r]]; while (cnt[s[r]] > 1) { --cnt[s[l++]]; } ans = max(ans, r - l + 1); } return ans; } };

代码解释：

  1. 变量初始化：
    • cnt[128]：用于记录每个ASCII字符在当前窗口中的出现次数，初始化为0。
    • ans：存储最长无重复字符子串的长度，初始为0。
    • n：字符串的长度。
  2. 滑动窗口：
    • 使用两个指针l（左）和r（右）表示窗口的左右边界。
    • r指针向右移动，扩展窗口，并增加当前字符的计数。
  3. 处理重复字符：
    • 如果当前字符的计数超过1，说明窗口中存在重复字符。
    • 移动左指针l，并减少左指针指向字符的计数，直到当前字符的计数恢复为1。
  4. 更新最大长度：
    • 每次窗口调整后，计算当前窗口的长度r - l + 1，并更新ans为最大值。
  5. 返回值：
    • 返回ans，即最长无重复字符子串的长度。

示例测试：

  • 输入"abcabcbb"，输出3（最长子串"abc"）。
  • 输入"bbbbb"，输出1（最长子串"b"）。
  • 输入"pwwkew"，输出3（最长子串"wke"或"kew"）。

该算法的时间复杂度为O(n)，空间复杂度为O(1)（因为cnt数组的大小固定为128）。

这段代码实现了一个函数 lengthOfLongestSubstring，用于找出给定字符串中最长的无重复字符的子串的长度。以下是代码的逐步解释：

1. 函数声明和变量初始化

  • cnt[128]：一个数组，用于记录当前窗口中每个ASCII字符的出现次数（ASCII字符共有128个）。
  • ans：初始化为0，用于保存最终的最长子串长度。
  • n：字符串 s 的长度。

2. 滑动窗口遍历字符串

  • 使用滑动窗口技术，l 和 r 分别表示窗口的左右边界。
  • r 从0开始向右移动，每次将 s[r] 的计数加1。

3. 处理重复字符

  • 如果当前字符 s[r] 的计数大于1，说明窗口中出现了重复字符。
  • 此时需要移动左指针 l，并减少 s[l] 的计数，直到 s[r] 的计数不再大于1（即窗口中不再有重复字符）。

4. 更新最大长度

  • 每次窗口调整后，计算当前窗口的长度 r - l + 1。
  • 如果当前窗口长度大于 ans，则更新 ans。

5. 返回结果

  • 最终返回 ans，即最长无重复字符子串的长度。

示例说明

假设输入字符串为 "abcabcbb"： 1. 初始时 l = 0, r = 0，窗口为 "a"，ans = 1。 2. r 移动到1，窗口为 "ab"，ans = 2。 3. r 移动到2，窗口为 "abc"，ans = 3。 4. r 移动到3，字符 'a' 重复，移动 l 到1，窗口为 "bca"，ans 仍为3。 5. 依此类推，最终最长无重复子串为 "abc" 或 "bca" 等，长度为3。

时间复杂度

  • 该算法的时间复杂度为 O(n)，其中 n 是字符串的长度。因为每个字符最多被访问两次（一次由 r，一次由 l）。

空间复杂度

  • 空间复杂度为 O(1)，因为 cnt 数组的大小固定为128（ASCII字符集的大小）。

class Solution { public: int climbStairs(int n) { int a = 0, b = 1; for (int i = 0; i < n; ++i) { int c = a + b; a = b; b = c; } return b; } };

这段代码实现了一个计算爬楼梯方法的函数，使用的是动态规划的思想。问题描述是：假设你正在爬楼梯，每次可以爬1个或2个台阶，问爬到第n个台阶有多少种不同的方法。

代码解释：

  1. 变量初始化：
    • a初始化为0，表示到达第0个台阶的方法数（可以理解为起始点）。
    • b初始化为1，表示到达第1个台阶的方法数（只有1种方法：爬1个台阶）。
  2. 循环计算：
    • 循环从i = 0到i = n-1，共迭代n次。
    • 每次迭代中：
      • c = a + b：计算到达当前台阶的方法数，等于前一个台阶的方法数（b）和前两个台阶的方法数（a）之和。这是因为你可以从第i-1个台阶爬1个台阶上来，或者从第i-2个台阶爬2个台阶上来。
      • 更新a和b：a取之前b的值，b取当前计算的c的值。这样在下一轮迭代中，a和b分别代表前两个和前一个台阶的方法数。
  3. 返回值：
    • 循环结束后，b中存储的就是到达第n个台阶的方法数，因此返回b。

示例：

  • 当n = 2时：
    • 第一次迭代：c = 0 + 1 = 1，a = 1，b = 1。
    • 第二次迭代：c = 1 + 1 = 2，a = 1，b = 2。
    • 返回2，即有两种方法（1+1或2）。
  • 当n = 3时：
    • 第一次迭代：c = 0 + 1 = 1，a = 1，b = 1。
    • 第二次迭代：c = 1 + 1 = 2，a = 1，b = 2。
    • 第三次迭代：c = 1 + 2 = 3，a = 2，b = 3。
    • 返回3，即有三种方法（1+1+1, 1+2, 2+1）。

本质：

这段代码实际上是计算斐波那契数列的第n+1项（如果斐波那契数列定义为F(0)=0, F(1)=1）。因为爬楼梯问题的方法数遵循斐波那契数列的递推关系。

优点：

  • 时间复杂度：O(n)，只需一次循环。
  • 空间复杂度：O(1)，只用了常数个额外空间。

代码解释：

  1. 变量初始化：
    • a 初始化为 0，表示爬 0 阶楼梯的方法数（初始条件）。
    • b 初始化为 1，表示爬 1 阶楼梯的方法数（初始条件）。
  2. 循环计算：
    • 循环从 i = 0 到 i = n-1，共迭代 n 次。
    • 每次迭代计算 c = a + b，即当前阶数的方法数等于前两阶方法数之和（斐波那契数列性质）。
    • 更新 a 和 b：
      • a 更新为上一轮的 b（即前一个阶数的方法数）。
      • b 更新为当前计算的 c（即当前阶数的方法数）。
  3. 返回值：
    • 循环结束后，b 中存储的是爬 n 阶楼梯的方法数，直接返回 b。

示例说明：

  • 如果 n = 3：
    • 第 1 次迭代：c = 0 + 1 = 1，a = 1，b = 1（对应爬 1 阶的方法数）。
    • 第 2 次迭代：c = 1 + 1 = 2，a = 1，b = 2（对应爬 2 阶的方法数）。
    • 第 3 次迭代：c = 1 + 2 = 3，a = 2，b = 3（对应爬 3 阶的方法数）。
    • 最终返回 3，即爬 3 阶楼梯有 3 种方法（1+1+1, 1+2, 2+1）。

特点：

  • 时间复杂度：O(n)，只需一次循环。
  • 空间复杂度：O(1)，仅用常数空间存储变量。
