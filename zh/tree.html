1. 树的基础概念

1.1 什么是树

树（Tree）是一种重要的非线性数据结构，由一个根节点和若干子树组成，具有分层关系。树是一种递归定义的数据结构，常用于表示具有层次关系的数据，例如文件系统、组织结构图、分类树等。

  • 树是一个由节点（Node）和边（Edge）组成的集合。
  • 树的特点：
    • 树中有且仅有一个称为根节点（Root）的节点，没有父节点。
    • 除根节点外的每个节点都有且仅有一个父节点。
    • 节点可以有零个或多个子节点。
    • 树中不存在环路（无回路）。

1.2 树的基本术语

以下是树结构中的一些基本术语：

  1. 节点（Node）：树的基本组成单位，每个节点可以存储数据。
  2. 根节点（Root）：树的顶层节点，没有父节点。
  3. 父节点（Parent）：一个节点的直接上层节点。
  4. 子节点（Child）：一个节点的直接下层节点。
  5. 兄弟节点（Sibling）：同一个父节点的子节点。
  6. 叶子节点（Leaf）：没有子节点的节点。
  7. 内部节点（Internal Node）：至少有一个子节点的节点。
  8. 路径（Path）：从一个节点到另一个节点的节点序列。
  9. 路径长度（Path Length）：路径中边的数量。
  10. 深度（Depth）：从根节点到某个节点的路径长度。
  11. 高度（Height）：从某个节点到其最远叶子节点的路径长度。
  12. 层次（Level）：树中节点的深度 + 1。
  13. 度（Degree）：一个节点拥有的子节点数量。
  14. 树的度：树中所有节点的最大度数。
  15. 子树（Subtree）：以某个节点为根的树。
1.3 树的分类

根据树的结构和性质，树可以分为以下几类：

  1. 普通树（General Tree）：
    • 每个节点可以有任意数量的子节点。
  2. 二叉树（Binary Tree）：
    • 每个节点最多有两个子节点，分别称为左子节点和右子节点。
  3. 完全二叉树（Complete Binary Tree）：
    • 除最后一层外，所有层的节点都达到最大数量，最后一层的节点从左到右连续排列。
  4. 满二叉树（Full Binary Tree）：
    • 所有非叶子节点都有两个子节点，且所有叶子节点在同一层。
  5. 平衡二叉树（Balanced Binary Tree）：
    • 任意节点的左右子树高度差不超过1。
  6. 二叉搜索树（Binary Search Tree, BST）：
    • 左子树的所有节点值小于根节点值，右子树的所有节点值大于根节点值。
  7. AVL树：
    • 一种严格平衡的二叉搜索树，保证左右子树高度差不超过1。
  8. 红黑树（Red-Black Tree）：
    • 一种自平衡二叉搜索树，节点被标记为红色或黑色，用于保证树的平衡性。
  9. B树和B+树：
    • 一种多路平衡搜索树，常用于数据库和文件系统。
  10. 字典树（Trie）：
    • 一种用于字符串存储和搜索的树结构。
  11. 森林（Forest）：
    • 由多棵互不相交的树组成的集合。
1.4 树的性质

树具有以下重要性质：

  1. 节点数与边数的关系：
    • 一棵有 (n) 个节点的树有 (n-1) 条边。
  2. 树的层次关系：
    • 树的第 (i) 层最多有 (2^{i-1}) 个节点（对于二叉树）。
  3. 二叉树的节点总数：
    • 高度为 (h) 的二叉树最多有 (2^h - 1) 个节点。
  4. 完全二叉树的性质：
    • 对于一棵有 (n) 个节点的完全二叉树，编号为 (i) 的节点：
      • 父节点编号为 (i/2 )。
      • 左子节点编号为 (2i)。
      • 右子节点编号为 (2i+1)。
  5. 树的高度与节点数的关系：
    • 高度为 (h) 的二叉树至少有 (h) 个节点，最多有 (2^h - 1) 个节点。
  6. 平衡性：
    • 平衡树的高度与节点数呈对数关系，保证了高效的查找、插入和删除操作。

树是一种灵活且高效的数据结构，广泛应用于算法设计、数据存储和信息检索等领域。

2.1 树的存储方式

树是一种重要的非线性数据结构，其存储方式主要有两种：链式存储和顺序存储。下面分别介绍这两种存储方式的特点、实现方法及其适用场景。

1. 链式存储

定义
链式存储是通过指针（或引用）将树的节点连接起来的一种存储方式。每个节点包含数据域和指针域，指针域用于指向其子节点或父节点。

实现方式
- 二叉树的链式存储
每个节点包含三个部分： 1. 数据域：存储节点的值。 2. 左指针域：指向左子节点。 3. 右指针域：指向右子节点。

  • 多叉树的链式存储
    对于多叉树，每个节点可以有多个子节点，因此可以使用一个链表或数组来存储子节点的指针。
    • 孩子链表法：每个节点的子节点用链表存储。
    • 孩子兄弟表示法：每个节点有两个指针，一个指向第一个孩子，一个指向下一个兄弟节点。

特点
- 节点之间通过指针连接，结构灵活。 - 节点的数量和形态不受限制，适合存储动态变化的树。 - 需要额外的存储空间来存储指针。

适用场景
- 树的结构不固定，节点数量动态变化。 - 需要频繁插入或删除节点的场景。

2. 顺序存储

定义
顺序存储是用一个数组来存储树的节点，节点之间的父子关系通过数组下标来表示。

实现方式
- 完全二叉树的顺序存储
对于完全二叉树，可以用一个数组存储节点，节点的下标与其父子关系满足以下规律： 1. 若节点的下标为 i，则： - 父节点的下标为 (i-1)/2。 - 左子节点的下标为 2*i + 1。 - 右子节点的下标为 2*i + 2。 2. 数组的第一个元素存储根节点。

  • 一般树的顺序存储
    对于一般树（非完全二叉树），顺序存储需要预留足够的空间来存储所有可能的节点，通常会浪费大量空间，因此不常用。

特点
- 节点之间的关系通过数组下标计算，访问效率高。 - 适合存储完全二叉树或接近完全二叉树。 - 空间利用率较低，尤其是对于稀疏树。

适用场景
- 树的结构固定，节点数量已知。 - 完全二叉树或接近完全二叉树的场景。

总结对比
存储方式 优点 缺点 适用场景
链式存储 结构灵活，适合动态变化的树 需要额外存储指针，访问效率较低 动态树结构，插入删除频繁
顺序存储 节点关系通过下标计算，访问效率高 空间利用率低，不适合稀疏或动态变化的树 完全二叉树或固定结构的树

根据具体应用场景选择合适的存储方式，可以有效提高程序的效率和资源利用率。

  • 前序遍历
  • 中序遍历
  • 后序遍历

    树的遍历是指按照一定的顺序访问树中每个节点的过程。根据访问节点的顺序不同，树的遍历可以分为以下几种方式：
1. 前序遍历（Preorder Traversal）

前序遍历的顺序是：根节点 → 左子树 → 右子树。

4. 层序遍历（Level Order Traversal）

层序遍历是按照树的层次，从上到下、从左到右逐层访问节点。

层序遍历通常使用队列来实现： 1. 将根节点入队。 2. 当队列不为空时： - 取出队首节点，访问该节点。 - 如果该节点有左子节点，将左子节点入队。 - 如果该节点有右子节点，将右子节点入队。

总结
  • 前序遍历：根 → 左 → 右
  • 中序遍历：左 → 根 → 右
  • 后序遍历：左 → 右 → 根
  • 层序遍历：逐层从左到右访问

这四种遍历方式是树的基本操作，理解并掌握它们对于树的操作和算法设计非常重要。

2.3 c语言 树的插入与删除

在C语言中，树是一种重要的数据结构，常见的树包括二叉树、二叉搜索树（BST）、平衡二叉树（如AVL树）、红黑树等。本文将以二叉搜索树（Binary Search Tree, BST）为例，介绍树的插入和删除操作。

  • 每个节点的值大于其左子树中所有节点的值。
  • 每个节点的值小于其右子树中所有节点的值。
